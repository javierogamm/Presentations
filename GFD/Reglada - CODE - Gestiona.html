Section

Condicionales

Section — Bloque condicional

Muestra u oculta un párrafo del documento o formulario según el valor de un campo del expediente.

Estructura base
{{#section_XXXX | condition : (personalized.ReferenciaTesauro == "REFVARIABLE") }} ——— PÁRRAFO CONDICIONADO ——— {{/section_XXXX}}

Let

Escritura de datos

Función Let — Asignación de valor

Permite asignar o calcular el valor de un campo tesauro en tiempo de tramitación, de forma imperativa y condicional.

Estructura base
{{ let | reference: personalized.CampoTesauro1 | result: (OPERACIÓN) }}

reference

Referencia del campo tesauro donde se escribe el resultado

result

Valor literal, referencia a otro campo, o expresión calculada

Gestiona CODE · Let
Let Booleano

Let — Booleanos

true / false
{{let | reference: personalized.CampoBooleano | result: true}} {{let | reference: personalized.CampoBooleano | result: false}}

Uso habitual dentro de condicionales

Se utiliza para marcar flags o estados dentro del expediente: activar/desactivar condiciones, señalar que una fase ha sido completada, o forzar un valor en un campo check/booleano.

Combinado con section
{{#section_MARCADO | condition: (personalized.CampoBooleano == true) }} Párrafo que aparece si el campo es verdadero {{/section_MARCADO}}

Circuitos de resolución

Circuitos de resolución — Simples

01

Circuito simple

Un único expediente pasa por el circuito de resolución. El resultado (concedido / denegado / caducidad) afecta a ese expediente individual. La firma y notificación van referenciadas a ese expediente.

↳

Flujo típico

Solicitud → Informe técnico → Propuesta de resolución → Firma → Notificación al interesado → Archivo.

Circuitos plurales

Circuitos de resolución — Plurales

N

Circuito plural

Permite resolver múltiples expedientes en un único acto resolutorio. Se agrupan por convocatoria, subvención o expediente matriz.

✓

Capacidades clave

Importar datos de N expedientes · Resolver masivamente · Notificar a múltiples interesados con documentos individualizados.

Gestiona · Formación23 / 27
Tramitación Reglada
Modelos de circuito plural

Modelos de resolución plural

1:N

Expedientes individuales

Cada expediente es independiente. El circuito plural los agrupa y resuelve en un único acto, generando notificaciones individuales.

M→N

Expediente matriz

Un expediente central gestiona la convocatoria y agrupa N expedientes asociados. La resolución aplica a todos los vinculados.
